## ЕСЛИ БУФЕР НЕ ПУСТОЙ

- Все блоки имеют отступы 2мм сверху и снизу
- Все блоки кроме информационного и URL имеют верхний border

### Блок Информационный блок
- Заполняет всё свободное пространство кроме других блоков, прижимая их вниз. Минимальная высота блока 3см
- Заголовок "Copied!", крупный, центрованный, зелёный
- Подзаголовок:
   - Зависит от последнего действия
   - Если copied:
      - "Copied to clipboard: WHAT_COPIED"
      - WHAT_COPIED = то, что скопировано в clopboard
      - Если WHAT_COPIED != nothing, то чёрная, жирная
      - Если WHAT_COPIED = nothing, то серая (как fallback)
      - При сохранении в clopboard нового значения, WHAT_COPIED меняется соответственно
   - Если saved:
      - "Downloaded: WHAT_DOWNLOADED"
      - WHAT_DOWNLOADED = последний скопированный файл, чёрный, жирный
      - При сохранении файла, WHAT_COPIED меняется соответственно
   - Если ничего не скопировано и не сохранено, то строки нет
   - Наличие/отсутствие подзаголовка не должно смещать положения других элементов

### Блок URL
- Inline:
   - Сам URL в поле наподобие того, что в блоке "Developer tools" с той же логикой цветов, обрезания, копирования. Заполянет ширину материнского элемента
- Кнопка external-link: открывает ссылку в новой вкладке

### Блок Copy
- Всё inline
- Содержит:
   - "Copy"
   - Кнопки "Text", "Markdown", "Image"

### Блок Download
- Содержит:
   - "Download"
   - Markdown
   - HTML
   - PNG
   - JPEG
- При нажатии на кнопку, скачивается соответствующий файл

### Developer tools
- Блок показывается только если включено в настройках пользователя
- Каждая запись:
   - Отдельной строкой
   - Формат: [название в качестве label строки] [поле для копирования]
   - Строка заполняет всю ширину окна, без переносов
   - Поле для копирования:
      - Выглядит как код
      - В конце есть иконка copy
      - Если строка не вмещается, то обрезается визуально, но по факту она там полностью
      - Текст выделить нельзя
      - Все поля для копирвоания равной ширины
      - Это поле тоже считаем кнопкой и подкрашиваем соответственно (обычное, недоступное, последнее копирование)
   - При нажатии на строку (включая label), значение копируется в буфер обмена

### Блок again
- Две кнопки inline "PICK ELEMENT" и "CAPTURE PAGE"
- Логика кнопок полностью совпадает кнопками страницы START

## СОСТОЯНИЯ КНОПОК

### Выделенная кнопка
- Если окно только открылось после копирования, то выделенной кнопкой является та, которая соответствует сохранённым в clipboard данным
- Далее выделенной яаляется последняя нажатая
- Если окно COPIED открылось пользователем повторно, то выделенной кнопки нет
- Единовременно может быть выделено не более 1 кнопки
- Выделяется зелёным
- При нажатии на кнопку:
   - Если кнопка копирования, то копирует в буфер обмена
   - Если кнопка скачивания, то скачивает файл

### Если для поля/кнопки нет данных в кэше
- Если для какой-то из опций нечего отдать пользователю то кнопка серая, без изменения состояний
- Учесть, что часть данных мы не храним в кэше, а вычисляем из других данных (которые в кэше)
- При нажатии на этот формат tooltip: "Nothing for this format"

### Если пользователь выключил формат
- Кнопка имеет стиль аналогичный как если нет данных в кэше
- Текст строки зачёркнут
- При нажатии на этот формат tooltip: "Turned off in the settings"
